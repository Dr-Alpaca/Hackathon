审计清单
安全性
- 文件上传是否存在路径穿越风险？文件名是否使用 uuid 或时间戳+随机字符串？
- 是否存在 SQL 注入风险（尤其是原生 SQL 拼接）？
- 是否限制了上传文件的类型和大小？是否验证了真实的 MIME 类型？
- 是否使用了硬编码的敏感信息（如数据库密码、API Key）？

业务逻辑正确性
- 发帖时自动创建店铺：是否检查了重复店铺（同一食堂+窗口名）？是否使用 get_or_create 模式？
- 用户行为表 (actions) 的唯一约束：插入时是否处理了 IntegrityError 并返回友好的错误提示？
- 推荐列表：是否过滤了无效数据（如 shop_id 不存在）？

异常处理与用户体验
- 关键接口是否有统一的错误返回格式？
- 第三方调用（如 LLM API）是否有超时控制和降级策略（规则兜底）？
- 是否有全局异常捕获（如 @app.errorhandler)？

可维护性
- 代码是否按功能拆分为多个模块（models.py, routes.py, utils.py）？
- 配置是否外部化（环境变量或 config 文件）？
- 是否有足够的注释和 docstring？
