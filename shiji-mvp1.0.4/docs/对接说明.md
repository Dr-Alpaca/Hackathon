## 前端同学 C - 对接任务

### 1. API 接口文档

**基础URL：** `http://localhost:5000`

#### 1.1 获取推荐列表

```
GET /api/recommendations
```

返回示例：
```json
{
  "code": 0,
  "message": "success",
  "data": [
    {
      "id": 1,
      "shop_id": 1,
      "canteen": "一食堂三楼",
      "shop_name": "7号窗口",
      "dish_name": "铁板黑椒鸡排饭",
      "quote": "鸡排是现煎的，黑椒汁给得足",
      "tags": ["出餐快", "量大"],
      "image_url": "/uploads/xxx.jpg",
      "created_at": "2024-01-01T00:00:00Z"
    }
  ]
}
```

#### 1.2 发帖（创建帖子）

```
POST /api/posts
Content-Type: multipart/form-data
```

参数：
- `text` (string): 帖子文字内容
- `images` (file): 图片文件，可多个（可选）
- `device_id` (string): 设备ID，前端生成存在localStorage

返回示例：
```json
{
  "code": 0,
  "data": { "post_id": 123 }
}
```

#### 1.3 获取店铺详情

```
GET /api/shops/<shop_id>
```

返回：店铺信息 + 该店铺下所有帖子（按时间倒序）

#### 1.4 记录用户行为（我想吃）

```
POST /api/actions
Content-Type: application/json
```

Body:
```json
{
  "device_id": "xxx",
  "target_type": "post",
  "target_id": 123,
  "action_type": "i_want_eat"
}
```

## 常见问题

**Q: LLM提取失败怎么办？**
A: 会自动降级到规则提取，不影响功能，只是准确率稍低。

**Q: 图片不显示？**
A: 检查 `uploads/` 目录是否存在，权限是否可写。

**Q: 唯一约束冲突？**
A: 同一用户同一天对同一帖子点"我想吃"会触发，这是正常设计，已做错误处理。
